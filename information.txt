üîÑ Workflow of the Offline Mesh Network System
Step 1 ‚Äî App Installation (Initial Internet Use)
The user first uses the internet only once to download the network application. After installation, the system primarily operates offline.
Step 2 ‚Äî Wallet Creation (User Identity)
Each user creates a wallet that acts as their unique username.
This wallet is based on Ethereum-style mnemonic phrases.
The mnemonic generates two cryptographic keys:
Public Key (for encryption / identification)
Private Key (for decryption / ownership)
Messages encrypted with one key can only be decrypted using its complementary key, ensuring secure communication.
Step 3 ‚Äî Connection to Pinging Server
After wallet creation, the application briefly connects to a Pinging Server.
This server:
Registers the wallet
Helps generate a unique network address called IPVX
Initializes the user into the mesh network
After this step, the offline system can function.
Step 4 ‚Äî IPVX Generation (Custom Offline Addressing)
A custom addressing protocol called IPVX is created, inspired by IPv6.
Core properties:
Hexadecimal based
Initial length: 16 bytes
Includes:
Prefix (1 byte): Distinguishes IPVX from standard IPv6
Suffix (binary): Used to identify and connect offline users
Each user is assigned a number from 0‚Äì255, converted into a binary byte and appended to the IPVX.
This makes the initial IPVX length 18 bytes.
Step 5 ‚Äî Network Expansion
When another user connects:
An additional byte is added to the IPVX
IPVX becomes 19 bytes
For every new user joining the mesh, the address grows sequentially by 1 byte
This creates a dynamic addressing structure that reflects network growth.
Step 6 ‚Äî Offline Mesh Communication
Using IPVX:
Devices connect directly in a peer-to-peer mesh
Each node can relay data for others
No central servers are required
Communication remains encrypted and decentralized
Because IPVX behaves similarly to IPv6, it inherits routing flexibility while operating entirely in an offline environment.
